本目录定义 ADPS 案例报告中使用的工程术语。每个条目说明机制、对应的失败方式与适用条件。当前定义源于 东方屹腾执行型 Agent 案例报告，在获得更多独立案例印证前保持“案例衍生”状态。

  • 控制平面与叙事平面：执行型 Agent 的二元论 — 先把活的不确定的收敛成死的可控的——控制平面机械化，叙事平面靠驾驭。
  • 执行型 Agent 不是内容生成型 Agent — 参数错一位会破坏业务数据的场景是执行型，文本松传递、可重试的是内容生成型。
  • 会话统一状态平面：三权分立 — 叙事/机械/调度三个平面各管一段：用户要什么做到哪了、API 参数从哪来对不对、先跑哪步跑完没有。
  • 机械状态平面与 Provenance 来源坐标 — 业务 API 入参不经 LLM 生成，每个参数有唯一、可审计、可 fail-fast 的机械来源坐标。
  • 叙事平面三件套：锚、账、集 — 锚定原始目标、账记关键里程碑、集投影出当前该看的工作集，防长链路目标漂移。
  • 草稿纸看板：ReAct 的工作记忆 — Thought/Action/Observation 链式追加，后续步骤读它沿已完成的事继续推理。
  • 记忆信封：每一步注入给模型的上下文包 — 每步封一只信封交给模型：锚 + 投影出的集 + 意图结论 + 召回经验 + 必要机械态。
  • L1/L2/L3 分层记忆与经验召回 — L3 存经验语义摘要 + 指向 L2 原始事实的 id，关键边界入口才召回，像内存分页换入换出。
  • Orchestrator 与 MessageHandler 门面 — 编排器是所有能力的家与控制流树干，门面专管 SSE 表演逻辑，控制与展示解耦。
  • 意图即编译：把活的编译成死的 — 把自然语言意图编译成离散控制信号，配 unknown 兜底，意图网关按信号编排下游。
  • 任务 DAG 与任务状态机 — 被 ReAct 跨步漏步逼出来，把执行顺序从模型判断里拿出来钉进任务图 + 状态机。
  • 可观测性：玻璃罩里的瑞士机械表 — 把 Agent 的精神活动与躯体活动封装成带语义事件，按时间轴对开发者完整可见。
  • HITL 阻塞-恢复：叙事态续作与机械态阻塞 — 关键操作执行前阻塞等人审，借任务状态机水到渠成，声明式配置挂人审点。
