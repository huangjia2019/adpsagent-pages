概念/概念定义

会话统一状态平面：三权分立

出自东方屹腾执行型 Agent 落地（案例提供 梁博）

概念定义

一次会话里的全部运行态上下文，不应混在同一处读写。会话统一状态平面把这些上下文拆成三个职责清楚的平面，每个平面只回答一个问题、只用一种数据形态、互不越界。这是东方屹腾执行型 Agent 最终稳定下来的主干设计。

使用说明

三权分立的意思是叙事、机械、调度三个平面各管一段。叙事状态平面用自然语言汇集"用户要什么、做到哪了"，它是 Agent 对世界和当前状况的感知理解，有些信息全量保存、有些压缩蒸馏、有些只在内存、有些落库甚至建向量索引。机械状态平面专管 API 参数字典，回答"参数从哪来、对不对"，存的是严格键值，每个参数都带来源坐标，可审计、可在出错时立即失败。任务调度状态平面只关心任务图里节点的顺序和状态，回答"先跑哪步、跑完没有"，下游节点能被启动的前提是上游全部完成、本节点就绪。三个平面用三种语言说三件事：进展用自然语言，参数用机读键值，调度用任务图。

它防住的失败，是这三件事一旦搅在一起就会反复出现的那一类。最早建调度上下文时，叙事内容混在里面，每个任务节点的结果详情也在同一处读写传递。结果是参数和进展互相污染：模型要从一段又有目标描述、又有中间思绪、又有 API 返回值的上下文里，自己分辨哪个是下一步该绑定的 id。在企业业务系统里，这种 id 往往是雪花算法生成的几十位随机字符串，让模型从混合上下文里原样取出再绑定，极不可靠——有时成功有时失败，什么时候失败既不可控也无法预料。把三件事分开之后，参数走机械平面、由程序机械传递不经模型生成，进展走叙事平面，调度走任务图，每条线各自独立，一条线上的漂移不会扩散到另一条。

东方屹腾的薪资组快速搭建是个清楚的切片。这个场景要先给企业原有薪资组数据建快照、确认快照后再导入匹配到的模板、导入失败用快照回滚。"先建快照再导入再回滚"这套顺序属于调度平面，由任务图的节点依赖钉死，不靠模型临场决定。模板匹配接口返回的 template_id，下游导入时必须原样作为入参，这属于机械平面，由参数坐标维护、机械传递。而"用户想搭一个什么样的薪资组、现在进行到哪一步"这类描述，属于叙事平面，用自然语言记录并在需要时投影注入。同一个任务执行过程里产生的副产物，按性质分别归到这三个平面，没有一处需要模型在混合上下文里替程序做判断。

何时需要

当你的 Agent 要在不可改造的业务接口上交付多步流程，每一步的参数都得有唯一、可审计、能在出错时立即失败的来源，而不是靠模型从上下文里猜。这套三分法的上游，是先把运行态拆成控制平面与叙事平面；机械平面这一段怎么保证每个参数的唯一来源，见机械状态平面与 Provenance 坐标；叙事平面自己怎么组织目标与进展，见锚、账、集。
