参与方式。

ADPS 面向有真实经验的工程师和组织开放——尤其是有大规模部署、强监管行业、或者 有较硬可靠性约束下的 agent 系统建设经验的同行。共同体本身保持小规模。 成员资格通过贡献获得，而不是通过申请。

三种贡献方式

1. 在 GitHub 上贡献模式

模式目录在 huangjia2019/agent-design-patterns。 可以开 issue 提议新模式、建议改名，或为某个模式补充来自真实生产代码的工程切片。欢迎 PR。

2. 案例研究和复盘

如果你有一个生产级 agent 系统遇到了模式问题（或者用模式解掉了问题），我们想读。 给一份短报告——什么拓扑、什么认知功能、哪里坏了、什么有效、成本形状长什么样。

3. Harness 实现

开源 Harness 工作——把模式变成可部署东西的运行时层——是整个生态的护城河。 如果你维护或贡献一个 agent runtime、框架或工作流引擎，我们对把它的能力映射到矩阵上很感兴趣。
