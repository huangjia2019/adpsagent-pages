立场

ADPS 共同体对 Agent 工程中若干争议话题的立场专文。每一篇都给出明确、以证据为据的判断——我们持什么立场、为什么、边界在哪。

  • 多 Agent 何时该用
    多 Agent 不是 Agent 工程的下一个 frontier，而是矩阵里一个有边界的设计选择——单 Agent + 工具集 + Harness 覆盖 70%+ 的生产场景。
  • Prompt Engineering 的位置
    Prompt 是 Harness 的一个输入，不是 architecture——它是 Agent 工程的 1/10，不是中心。
  • 治理与 Safety
    治理不是外挂的基础设施，而是设计的一部分——OWASP Agentic Top 10 与八条原则精确映射。
  • Eval 的工程化
    Evaluation 是设计本身——pre-design / pre-prod / production 三段评估，缺一段系统就飘。
