Agent Design Patterns Society

A public pattern language for production AI agent systems.

ADPS is an independent research community developing shared technical references for production AI agent systems. It brings together software design research, open-source harness engineering, and attributed enterprise field reports, then turns recurring problems into pattern specifications that engineers can compare, test, and cite.

The 28-core-pattern matrix · 7 cognitive functions × 6 execution topologies

What we do

ADPS publishes three connected sets of technical references:

  • Pattern white papers — 28 core patterns and 6 extension or candidate specifications, each stating its problem, mechanics, applicability, known failure modes, and verification criteria.
  • Engineering blue books — attributed field cases that record how pattern combinations behave under real organizational and system constraints, with evidence and transfer boundaries stated explicitly.
  • Open reference assets — papers, selection methods, implementation examples, and engineering concepts that teams can inspect and adapt.

The framework

The two-axis framework — cognitive function × execution topology — was introduced in A Two-Dimensional Framework for AI Agent Design Patterns (Huang & Zhou, arXiv:2605.13850) and developed further in the Manning book Designing AI Agents. ADPS now stewards the catalog as a community asset.
